Что выведет программа? Объяснить вывод программы.

```go
package main

// Объявляем тип customError, который удовлетворяет интерфейсу builtin.error.
type customError struct {
	msg string
}
// Error метод реализует интерфейс builtin.error.
func (e *customError) Error() string {
	return e.msg
}

func test() *customError {
	{
		// do something
	}
	// nil — предварительно объявленный идентификатор, представляющий нулевое значение для типов
    // указатель, канал, функция, интерфейс, карта или срез.
	return nil
}

func main() {
	var err error
	err = test()
	// Условие выполняется потому что мы сравниваем интерфейс с nil. Интерфейс будет nil только тогда когда его поля будут nil.
	// А мы передали в него тип *customError и со значением равным nil.
	if err != nil {
		println("error")
		return
	}
	println("ok")
}
```

Ответ:
```
Программа выведет
error
Функция test() возвращает ссылку на структуру реализующую интерфейс error со значением nil. Переменная err имеет тип 
интерфейса error, и принимает значение ссылки на структуру customError реализующей интерфейс error.
В условии мы сравниваем с nil значение типа интерфейс err которое не может равняться nil потому что:
type iface struct {
    tab  *itab
    data unsafe.Pointer
}
Где tab это указатель на Interface Table или itable — структуру, которая хранит некоторые метаданные о типе и список
методов, используемых для удовлетворения интерфейса.
data — указывает на фактическую переменную с конкретным (статическим) типом.
err состоит из двух поинтеров, первый указывающий на itable для конкретной пары (статический тип type(customError),
интерфейс error), а data на копию оригинального значения customError{}. В err мы передали значение nil ссылки на тип
customError соответственно поле itab хранит метаданные о типе, а поле data равняется nil.
```
